自学计算机到极致

存储器

  • 内存
    • 电子产品设计的一个准则是防呆
    • 内存条，内存芯片

内存芯片

外形：一个长方块，上方有个缺口（凹陷），两边有引脚
缺口边两引脚距离：15.24mm(公制单位), 0.6inch(英尺), 600mil(密尔)
缺口邻边两引脚距离：2.54mm(公制单位), 0.1inch(英尺),100mil(密尔)

封装(机械参数，Mechanical Parameter)：外形与尺寸的合成

DATA SHEET（说明书，数据纸）：芯片的所有的参数（机械，电气参数…）都可以在这里查到，每个芯片都会有，一般为电子的，pdf。

引脚向外撇，呈八字的原因：电子产品在工厂进行装配时，都是自动化的，机器手抓两边时正好把引脚捏直了。

规律：引脚的标号， 缺口左边的引脚为1， 左下引脚为14, 右下引脚为15， 右上引脚为28。
从缺口左面开始逆时针转一圈到右边（按字母U的书写顺序）

芯片表面的字：
以视频为例：HM6264ALP-15
HM6264ALP-15为芯片的型号，HM是芯片的生产厂商的代号，6264是芯片真正的型号（相同型号可以换着使用），A是版本信息，L可能是低耗电，P未知，15芯片的性能（存取过程的时间）

芯片：黑块为树脂（耐热塑料）， 内部有一个硅片，电路肉眼看不见，肉眼可以看见一些触点，这些触点通过金属导线(黄金只有一点，引脚会延伸到硅片附近)连接到引脚，引脚是铜的

28引脚接电源正极，14接电源负极，一般是右上接正极，左下接负极。具体看说明书

电气参数（Electrical Parameter）：

  • 静态 DC(直流)
  • 动态 AC(交流)
  • 极限

DC operating conditions 直流参数
supply voltage 电源电压 min typ max

引脚1：NC(No Connection) 无连接引脚， 硅片与引脚没有连接, 有时可以与引脚28连接作为正极，电路板实现。

I/O 1 ~ 8：输入输出(input output), 数字通过I/O进行传输, 接到高电平为1，低电平为0

芯片内部有I 和 O 通过一个开关来连接I/O 通过一个信号

$\overline{\text{we}}$

电源负端就是”地”, 就是0V

拓展知识

电路中电源负极为0电压，电压的参考点 Grpimd(地面，平地)接地点 电压零点。电路中在接入一个电源，正极接在0点，负极的电压就为负数

集成电路的尺寸标注的最小单位为0.001inch(英尺) = 1mil(密尔)

关于sublime text的疑问

在写洛谷P2249 【深基13.例1】查找时发现了一个关于sublime text 的一个问题。也有可能是sublime的插件Cpp​Fast​Olympic​Coding的问题，或者是gcc，我并没有考证，只是发现了这个问题. 2023/8/4 23:50

当数组a位于main函数里时，sublime text 并不会给你输入的机会，直接结束输入，输出空白。
而将数组a置于main函数外时， 则可以正常输入。
数组a无论在main里，还是在main外都是可以通过洛谷测试。

二分查找

整数二分

假设有n个元素的数组， 数组下标从0开始, 到n-1结束，设前k个元素为蓝色，后n-k个为红色，k为未知数。开始时，数组全为灰色。根据条件来判断k的值，即蓝红边界。

朴素算法

设计一个蓝色指针指向数组的最左面， 随后向右移动， 找到蓝红边界。(蓝红边界条件由题目分析可得)，或设计一个红色指针指向数组的最右面， 随后向左移动， 找到蓝红边界。
算法复杂度是O(n)

二分查找

朴素算法之所以低下， 是因为它是一个元素一个元素的移动。
二分查找则在开始是查看数组中间的元素(蓝红指针的中间)的颜色，若为蓝色，则说明这个元素的前面都是蓝色，即可将左指针执行这个元素, 右指针同理。不断重复这种操作，即可得到蓝红边界
蓝色区域的元素可以说明都具有某种性质，列如: 小于等于某个数。

一般流程

  1. 建模：划分蓝红区域， 确定IsBlue()
  2. 确定返回 l 还是 r
  3. 套用算法模板
  4. …

0成本搭建博客

下载git

除了设置安装路径外，其他可以默认

命令行git -v验证

下载Node.js

同样出来安装路径，其余默认即可

命令行node -v验证

安装hexo

git bash 或 命令行输入 npm install -g hexo-cli

git bash 或 命令行输入 hexo -v

注册github账号

新建仓库

new repositor
Respository name 格式为 账号的名字.github.io

其余可以默认即可。

配置ssh免密登录

删除c:\Users\用户名.ssh文件夹

在c:\Users\用户名 文件夹用git bash 打开,输入 ssh-keygen -t rsa -C github邮件地址, 生成.ssh秘钥，输入后连敲三次回车

生成了一个新的 C:\Users\用户名.ssh文件夹，打开这个文件夹，找到.ssh\id_rsa.pub，打开并复制里面的内容

打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key，把复制的内容粘贴进去，title随便填

git bash 中输入ssh -T git@github.com 不用改

如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到：

Hi XXXX! You’ve successfully authenticated, but GitHub does not provide shell access.

看到这个信息说明SSH已配置成功

git config –global user.name 你的github账户名
git config –global user.email 填写你的github注册邮箱

使用hexo

在电脑随便找个地方，git bash 打开 输入 hexo init
会自动生成一些文件。

输入hexo g 生成，hexo就会在public文件夹生成相关html文件
输入hexo s 启动服务，开启本地预览服务

打开浏览器访问 http://localhost:4000 即可看到内容

按住ctrl, 鼠标点击git bash中的链接，即可浏览器打开

将博客部署到 github 个人主页

在博客的根目录下 git bash打开

输入npm install hexo-deployer-git --save

在根目录下的_config.yml 中末尾 改为

在博客根目录使用git bash打开，输入hexo d

https://用户名.github.io/ 即可打开

如果没有问题的话，需要等一会。
