Part 1: Introduction to Lexical Scanning

the job of the scanner is to identify the lexical elements, or tokens, in the input language.

lexcial scanner 词汇扫描器 识别词汇元素，tokens

We will start with a language that has only five lexical elements:

  • the four basic maths operators: *, /, + and -
  • decimal whole numbers which have 1 or more digits 0 .. 9

可以识别五个词汇元素，分别是四种基本数学运算符号，十进制数

Each token that we scan is going to be stored in this structure (from defs.h):

对于每个需要扫描的token 都将存储在这个结构体中， 用来分辨是符号还是整数，如果是整数的话，intvalue就是他的值。

where the token field can be one of these values (from defs.h):

When the token is a T_INTLIT (i.e. an integer literal), the intvalue field will hold the value of the integer that we scanned in.

token 的 值可以是 T_PLUS, T_MINUS, T_STAR, T_SLASH 分别对应 +， -， *， /，T_INTLIT则表示这个数是一个整数。

Functions in scan.c

The scan.c file holds the functions of our lexical scanner. We are going to read in one character at a time from our input file. However, there will be times when we need to “put back” a character if we have read too far ahead in the input stream. We also want to track what line we are currently on so that we can print the line number in our debug messages. All of this is done by the next() function:

static表示这个函数只在这个文件有效.

next()中, 由于Putback 是 extern 的全局变量, 所以初始值为 0 , 如果 Putback 不为0时,则表示读取文件指针多读了一个, 这是由于下文scanint(int c)导致的, 这个函数的功能是扫描所有整数并求出正确的整数值. 若Putback为0, 则说明指针位置正确, 如果的得到了 '\n', 则说明这行读取完毕, 行数加一.

Putback, intvalue, Infile都定义defs.h中.

Integer Literal Values

isdigit() 需要头文件 ctype.h.

如果既不是四个数学符号，也不是数字，则这个词汇扫描器无法识别，直接报错退出，输出未知字符，所在行数。

如果是数字则会调用scanint()来得到这个数，并将这个数的token，设为T_INTLIT.

Once we hit a decimal digit character, we call the helper function scanint()
with this first character. It will return the scanned integer value. To
do this, it has to read each character in turn, check that it’s a
legitimate digit, and build up the final number. Here is the code:

We start with a zero val value. Each time we get a character
in the set 0 to 9 we convert this to an int value with
chrpos(). We make val 10 times bigger and then add this new
digit to it.
For example, if we have the characters 3, 2, 8, we do:

  • val= 0 * 10 + 3, i.e. 3
  • val= 3 * 10 + 2, i.e. 32
  • val= 32 * 10 + 8, i.e. 328
    Right at the end, did you notice the call to putback(c)?
    We found a character that’s not a decimal digit at this point.
    We can’t simply discard it, but luckily we can put it back
    in the input stream to be consumed later.
    You may also ask at this point: why not simply subtract the ASCII value of
    ‘0’ from c to make it an integer? The answer is that, later on, we will
    be able to do chrpos("0123456789abcdef") to convert hexadecimal digits
    as well.
