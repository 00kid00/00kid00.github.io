Build Your Own Text Editor

setup

main()函数

创建一个新文件名字为 kilo.c, 写一个main()函数（kilo 就是这个文本编辑器的名字）

kilo.c:

在c, 你必须将可执行代码放入main()函数中， main()是特殊的函数， 当你的程序开始运行时，main()函数是默认的起点，执行完return后，程序退出，并返回一个整数给操作系统，返回0表示成功。

c 是一个编译型语言，需要通过 c 编译器，将其转化为可执行文件。

在shell中执行 cc kilo.c -o kilo 即可编译 kilo.c, 如果没有出错， 将得到一个叫 kilo 的可执行文件。
-o 代表 “输出”, 指定输出的可执行文件的名字为kilo.

输入（键入） ./kilo 后按下Enter可以运行 kilo, 这个程序没有答应字符，也没有输出，但通过echo $?可以检查退出状态（mian()的返回值） , 这个程序应该会返回0。

使用make编译

每次输入 cc kilo.c -o kilo 很累， make 程序 可以简单运行make来编译程序， 必须提供一个Makefile告诉他如何编译程序

创建一个名叫 Makefile 的文件， 输入以下内容：

第一行 kilo 是将要创建文件名字， kilo.c 是创建其所依赖的文件。 第二行要表明 kilo.c 转换为 kilo 的命令。第二行的开头必须是一个 tab character(tab键), 不能是空格。

编译命令中额外添加的内容:

  • $(CC) 是一个变量， make 默认将其扩展为 cc.
  • -Wall 是 “All Warning”, 显示所有的警告信息。程序在技术上没有问题，但会被编译器认为有问题是会发出警告，比如在初始化变量前使用变量。
  • -Wextra 和 -pedantic 打开更多的警告，在这个给教程中，只会回警告”unused variab”, 若出现其他前检查是否有错。
  • -std=c99 指定使用c语言标准的版本，C99允许将变量在函数内任何位置定义， 但ANSI C只允许将变量定义在函数开头。

现在我们有了 Makefile， 可以使用 make 来编译程序

如果输出 make: 'kilo' is update to data.. 表明kilo.c 最后修改文件的时间戳小于kilo. 这样的好处是多文件编译时，如果有的文件没有修改，则可以不用编译其。

进入原始模式(raw mode)

读取用户输入

kilo.c

read()和STDIN_FILENO来自于<unistd.h>, read()会从c中一个字节一个字节的读取， 知道读取到文件末尾，read() 返回所读字节数， 返回 0 表明达到文件末尾

有关 read() 与 STDIN_FILENO(或自行百度):

  • https://blog.csdn.net/super828/article/details/118770845
  • https://blog.csdn.net/sinat_25457161/article/details/48548231

编译后运行./kilo, 终端会与标准输入连接， 键盘的输入可以读到变量c中。

终端默认是 规范模式(canonical mode) , 也叫熟模式(cooked mode), 这种模式下只有个按下Entern后才会发送给程序， 这种模式有利于输入一行文本， 当输入错后可以使用Backspace 来修改错误， 之后按下 *Entern 发送给程序。但对于复杂的用户界面来说不能很好的工作。我们希望每次按下都会处理，这样可以做到立刻相应。

我们很难将终端设置为原始模式， 原始模式需要关闭终端中大量的标志。后面会一步一步将其关闭。

Ctrl-D 可以告诉 read() 已经到达了文件末尾。或者Ctrl-C 用信号立即终止进程

按 q 退出

为了证明规范模式如何工作，使用 q 退出， 程序将

kilo.c

输入一行包括 q, 然后输入 Enter, 程序将一个一个字符读取直到读取到 q, while循环将停止， 程序将退出， 在输入队列中， q之后的字符都将保持未读取的状态， 你可以看到程序退出后， 将会喂给shell

关闭回显

我们可以设置终端的属性通过：

  1. 使用tcgetattr()将当前的属性读取到一个结构体中
  2. 手动修改结构体
  3. 通过修改结构体， 使用tcsetattr()来修改终端属性

使用这种方式来关闭回显。

kilo.c:

struct, termios, tcgetattr(), ECHO, TCSAFLUSH都来自于<termios.h>

ECHO的功能是将你输入的字符都打印到终端中， 在原始模式下呈现用户界面是会有问题， 所以需要关闭。这个程序做相同的事情，只是不会打印你输入的字符。

程序退出后，你会发现你的终端也无法回显你的输入， Ctral-C开启新的输入行，输入reset, 按下Enter，大多情况下会将终端重置为正常状态，如过不行，重启重点模拟器， 下一步将解决这个问题。

终端属性可以通过tcgetattr()读入到termios结构体中， 修改结构体后， 可以使用tcsetattr()将其应用于终端。TCSAFLUSH指定何时更改应用，他等待所有的待处理数据写入终端， 并丢弃未读取的输入。

c_lflag字段是 “local flags”, macOs的<termios.h>的一条评论将其描述为”(dumping ground for other state)其他状态的垃圾场”, 也许他应该被认为是”杂项标志(“miscellaneous flags)”, 其他标志字段有: c_iflag(intput flags), c_oflag(output flags), c_cflag(control flags), 我们必须修改这些来启用原始模式。

ECHO是一个 bitflag, 二进制数被定义为 0000 0000 0000 0000 0000 0000 000 01000， 我们对其使用bitwise-NOT操作符（~）得到 1111 1111 1111 1111 1111 1111 1111 0111, 然后将这个变量标志字段bitwise-AND，强迫flags字段第四位变成0, 其他位保留当前的值。 这样的翻转位在c中很常见。

退出禁用原始模式

当退出时，恢复原来的属性。我们将原始状态保存termios结构体中, 但程序退出后， 使用tcsetattr()将其应用到终端中。

kilo.c:

atexit() 来自于 <termios.h>, 我们使用他去注册我们disableRawMode()函数，当我们的程序退出时，自动的调用这个函数。

我们将原始的终端属性存储在全局变量orig_termios中， 将orig_termiso拷贝到raw中, 之后修改 raw.

你可以注意到但退出程序时剩下的输入不再喂给shell, 这是因为程序退出时TCSAFLUSH选项，被传递给tcsetattr()中， 更改应用到终端之前， 他丢弃了所有未读取的输入;

关闭规范模式

ICANON标志允许我们关闭规范模式， 这表示我们的输入会一个字节一个字节的读取， 而不是一行一行的读取。

kilo.c

ICANON来自于<termios.h>, 输入标志（c_iflag中的标志）通常以 I开头，就像 ICANON, 然而 ICANON 不是输入标志， 他是一个 “local” 标志，在 c_iflag字段.

现在程序一旦按下q便会推出。

显示按键

为了更好的了解原始模式下的输入如何工作， 让我们打印每个字节read(), 我们打印每个字符的ASCII值，以及他所代表的字符（如果他是可打印的字符）

kilo.c

isctrl() 来自于 <ctype.h>, printf() 来自于 <stdio.h>

isctrl() 测试一个字符是否时控制字符，控制字符是不可打印的字符，我们不想打印到屏幕上，ASCII代码 0 - 31 是所有控制字符， 127也是一个控制字符，ASCII代码32 - 126 都是打印的。

printf() 可以打印一个字符的多种形式，%d表示将字节格式转化为十进制模式（ASCII代码）, %c直接作为字符写出字节。

这个程序向我们展示各种按键如何转换我们读取的字节， 大多数普通按键直接转换为他们所代表的字符。

  • 方向键， Page Up, Page Down, Home, and End 所有输入3个或4个字节到终端， 27, [, 之后有一个或两个字节。这称为转义序列。所有的转义序列都开始与于27字节。按下Escape(ESC)发送一个单独的27字节信号。
  • Backspace 是 字节127, Delete 是4字节的转义字符。
  • Enter 是字节10, 是新行字符， 也是’\n’.
  • Ctral-A 是1， Ctral-B是2, Ctral-C终止程序，但确实有效的Ctral组合键似乎将字母 A-Z 映射为 1-26.

如果你发现按下 Ctral-S 你的程序会被冻结，停止向程序发送你的输出。按下 Ctral-Q 告诉他继续发送输出。

按下 Ctral-Z 或 Ctral-Y，你的程序会被暂停在后台， 运行fg会将其带会前台（wsl中带会前台后一行一行读入）

关闭 Ctral-C 和 Ctral-Z 信号

默认情况下， Ctral-C 发送一个 SIGINT 信号给当前进程， 导致他终止。Ctral-Z 发送一个 SIGTSTP 信号给当前的进程， 是他暂停。让我们关闭这两个信号发送。
